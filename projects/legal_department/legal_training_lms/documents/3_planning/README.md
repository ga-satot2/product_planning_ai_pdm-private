# 法務研修LMSシステム ドキュメント

## ドキュメント一覧

### 1. システム設計書
- **ファイル**: `法務研修LMSシステム設計書.md`
- **内容**: システム概要、要件定義、データモデル設計
- **対象**: 開発チーム、プロダクトマネージャー

### 2. シーケンス図
- **ファイル**: `法務研修LMS_シーケンス図.md`
- **内容**: システムフローの詳細なシーケンス図
- **対象**: 開発チーム、アーキテクト

### 3. 予約変更機能設計
- **ファイル**: `予約変更機能_対策設計.md`
- **内容**: 予約変更機能の詳細設計と対策
- **対象**: 開発チーム、QAチーム

### 4. アーキテクチャ設計
- **ファイル**: `GoogleワークスペースLMS_アーキテクチャ設計.md`
- **内容**: Googleワークスペース制約下でのアーキテクチャ設計
- **対象**: アーキテクト、開発チーム

### 5. 実装例
- **ファイル**: `GoogleワークスペースLMS_実装例.md`
- **内容**: 具体的な実装コード例とテンプレート
- **対象**: 開発チーム

## ドキュメントの読み方

### 開発者向け
1. **アーキテクチャ設計** → システム全体の構成を理解
2. **実装例** → 具体的なコード実装を参考
3. **シーケンス図** → 処理フローを詳細に把握

### プロダクトマネージャー向け
1. **システム設計書** → 要件と機能を確認
2. **予約変更機能設計** → 重要な機能の詳細を理解
3. **アーキテクチャ設計** → コストと技術的制約を把握

### QAチーム向け
1. **シーケンス図** → テストシナリオの作成
2. **予約変更機能設計** → エラーケースの理解
3. **実装例** → テストデータの準備

## レビュープロセス

### 1. 技術レビュー
- **対象**: アーキテクチャ設計、実装例
- **レビュアー**: シニア開発者、アーキテクト
- **観点**: 技術的妥当性、パフォーマンス、セキュリティ

### 2. ビジネスレビュー
- **対象**: システム設計書、予約変更機能設計
- **レビュアー**: プロダクトマネージャー、経営陣
- **観点**: 要件充足、コスト、スケジュール

### 3. ユーザビリティレビュー
- **対象**: シーケンス図、実装例
- **レビュアー**: UXデザイナー、実際の利用者
- **観点**: 操作性、ユーザーエクスペリエンス
