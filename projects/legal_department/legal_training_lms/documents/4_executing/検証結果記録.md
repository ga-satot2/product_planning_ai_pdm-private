**自動実行の範囲**: clasp によるデプロイ確認・clasp run、**Playwright**（`run_webapp_verify_playwright.js` で home/courses/mypage/register）、**Playwright 拡張**（`run_webapp_verify_extended_playwright.js` で admin ページ・予約フロー E2E）を実行済み。2026-02-19 拡張 Playwright 再実行: admin Pass、予約フローはスキップ（同上）。認証は `~/.playwright_chrome_profile`。予約フローはコース・セッションが無い環境ではスキップ。「要手動確認」は実施後に Pass/Fail に更新可能。

**進捗が止まる理由（情報不足ではない）**: `clasp run getSystemStats` で現状は `totalCourses: 0`, `totalParticipants: 0` と取得済み。スプレッドシートにコース・セッションが 1 件も無いため、Web アプリに「予約する」リンクが表示されず、予約 E2E を実行できない。進めるには **[データマイグレーション手順](development/prototypes/docs/データマイグレーション手順.md)** に従い、グループ一覧・コース一覧・予約一覧・参加情報を投入したうえで `run_webapp_verify_extended_playwright.js` を再実行する。その他の要手動確認（Slack、Calendar、デプロイのアクセス設定）は、GAS のスクリプトプロパティや組織側の設定に依存し、手動確認または GAS エディタでの実行が必要。

## 4.1 Web アプリ（GAS デプロイ）

| 項目 | 結果 | 備考 |

| Web アプリがデプロイされ、ウェブアプリの URL が取得できる | Pass | clasp deployments で確認。getSystemStats / getUserEmail / formatDate を clasp run で実行済み |
