# 引き渡しチェックリスト

**作成日**: 2025-12-26  
**目的**: 法務部門への引き渡し時の確認事項

---

## 📋 引き渡し前の確認事項

### 1. データ準備

- [ ] [データマイグレーション手順](development/prototypes/docs/データマイグレーション手順.md) に従い、グループ一覧・コース一覧・予約一覧・参加情報が投入済みである（空の場合は先に実施）
- [ ] イベント一覧が正しく登録されている
- [ ] 登録者名簿が最新である
- [ ] 第3期生の出欠予約データが最新化されている
- [ ] データの整合性が確認されている

### 2. Slack設定

- [ ] 1期生チャンネルにIncoming Webhookが追加されている
- [ ] 2期生チャンネルにIncoming Webhookが追加されている
- [ ] 3期生チャンネルにIncoming Webhookが追加されている
- [ ] スクリプトプロパティに各グループのWebhook URLが設定されている
- [ ] スクリプトプロパティに各グループのチャンネルIDが設定されている
- [ ] 各グループのリマインド通知が正常に送信されることを確認

### 3. システム設定

- [ ] Google Calendar IDが正しく設定されている
- [ ] スプレッドシートIDが正しく設定されている
- [ ] すべてのスクリプトプロパティが正しく設定されている

### 4. 機能確認

- [ ] 研修コースの登録が正常に動作する
- [ ] カレンダーイベントの自動作成が正常に動作する
- [ ] 予約フォームの再生成が正常に動作する
- [ ] リマインド通知の送信が正常に動作する
- [ ] 参加情報の自動更新が正常に動作する

#### 4.1 Web アプリ（GAS デプロイ）

- [ ] Web アプリがデプロイされ、ウェブアプリの URL が取得できる
- [ ] デプロイの「アクセスできるユーザー」が組織内の全員（または利用範囲）に設定されている
- [ ] ホーム・研修コース・予約・マイページが表示・操作できる
- [ ] Web からの予約作成・キャンセルが正常に動作する
- [ ] 受講者登録（?page=register）が利用できる
- [ ] 管理者メールで管理者画面（統計・コース管理・セッション管理・一斉メール・レポート）にアクセスできる

### 5. ドキュメント

- [ ] 利用開始マニュアルが作成されている（Web アプリの利用手順・初回アクセス時の権限設定を含む）
- [ ] データインポート手順書が作成されている
- [ ] Slack App設定手順書が作成されている
- [ ] 引き渡しチェックリストが作成されている

## 📝 引き渡し時の説明事項

上記項目はすべて「法務部門向け利用開始マニュアル」に記載あり。引き渡し時に該当章を説明すること。（2026-02-19 説明事項最終確認済み）

### 1. システム概要

- [ ] システムの目的と機能を説明
- [ ] システム構成を説明
- [ ] 主要な機能の使い方を説明

### 2. 操作方法

- [ ] Web アプリの URL の共有と、初回アクセス時「アクセス権が必要です」が出た場合の依頼先（デプロイのアクセス設定）を説明
- [ ] 研修コースの登録方法を説明
- [ ] 予約フォームの再生成方法を説明
- [ ] リマインド通知の送信方法を説明
- [ ] 参加情報の更新方法を説明

### 3. トラブルシューティング

- [ ] よくある問題と解決方法を説明
- [ ] エラー発生時の対応方法を説明
- [ ] サポート体制を説明

### 4. メンテナンス方針

- [ ] **定期的なメンテナンスは不要**であることを説明
- [ ] 問題発生時の対応方法を説明
- [ ] 連絡先を伝達

- [ ] Web アプリの URL でログインし、ホーム・コース一覧・予約・マイページを利用できる
- [ ] 研修コースの登録ができる
- [ ] 予約フォームの再生成ができる
- [ ] リマインド通知の送信ができる
- [ ] 参加情報の更新ができる

### 開発側の確認

- [ ] すべての設定が完了している
- [ ] すべての機能が正常に動作している
- [ ] ドキュメントが整備されている
- [ ] サポート体制が確立されている
