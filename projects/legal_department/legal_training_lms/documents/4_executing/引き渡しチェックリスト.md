# 引き渡しチェックリスト

**作成日**: 2025-12-26  
**目的**: 法務部門への引き渡し時の確認事項

---

## 📋 引き渡し前の確認事項

### 1. データ準備

- [ ] イベント一覧が正しく登録されている
- [ ] 登録者名簿が最新である
- [ ] 第3期生の出欠予約データが最新化されている
- [ ] データの整合性が確認されている

### 2. Slack設定

- [ ] 1期生チャンネルにIncoming Webhookが追加されている
- [ ] 2期生チャンネルにIncoming Webhookが追加されている
- [ ] 3期生チャンネルにIncoming Webhookが追加されている
- [ ] スクリプトプロパティに各グループのWebhook URLが設定されている
- [ ] スクリプトプロパティに各グループのチャンネルIDが設定されている
- [ ] 各グループのリマインド通知が正常に送信されることを確認

### 3. システム設定

- [ ] Google Calendar IDが正しく設定されている
- [ ] スプレッドシートIDが正しく設定されている
- [ ] すべてのスクリプトプロパティが正しく設定されている

### 4. 機能確認

- [ ] 研修コースの登録が正常に動作する
- [ ] カレンダーイベントの自動作成が正常に動作する
- [ ] 予約フォームの再生成が正常に動作する
- [ ] リマインド通知の送信が正常に動作する
- [ ] 参加情報の自動更新が正常に動作する

### 5. ドキュメント

- [ ] 利用開始マニュアルが作成されている
- [ ] データインポート手順書が作成されている
- [ ] Slack App設定手順書が作成されている
- [ ] 引き渡しチェックリストが作成されている

---

## 📝 引き渡し時の説明事項

### 1. システム概要

- [ ] システムの目的と機能を説明
- [ ] システム構成を説明
- [ ] 主要な機能の使い方を説明

### 2. 操作方法

- [ ] 研修コースの登録方法を説明
- [ ] 予約フォームの再生成方法を説明
- [ ] リマインド通知の送信方法を説明
- [ ] 参加情報の更新方法を説明

### 3. トラブルシューティング

- [ ] よくある問題と解決方法を説明
- [ ] エラー発生時の対応方法を説明
- [ ] サポート体制を説明

### 4. メンテナンス方針

- [ ] **定期的なメンテナンスは不要**であることを説明
- [ ] 問題発生時の対応方法を説明
- [ ] 連絡先を伝達

---

## ✅ 引き渡し完了確認

### 法務部門側の確認

- [ ] システムの基本的な操作方法を理解している
- [ ] 研修コースの登録ができる
- [ ] 予約フォームの再生成ができる
- [ ] リマインド通知の送信ができる
- [ ] 参加情報の更新ができる

### 開発側の確認

- [ ] すべての設定が完了している
- [ ] すべての機能が正常に動作している
- [ ] ドキュメントが整備されている
- [ ] サポート体制が確立されている
