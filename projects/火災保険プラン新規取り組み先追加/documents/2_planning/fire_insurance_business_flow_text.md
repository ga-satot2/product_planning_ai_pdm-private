# 1. インシュアランス 火災保険業務

シーケンス図「インシュアランス 火災保険業務」をテキスト化したもの。

## 参加者

| 名称 | 説明 |
|------|------|
| SUPPLIER | 外部 |
| GAJ | 社内（保険会社または中核部門） |
| 代理店 | 保険代理店（仲介） |
| AGENT（詳細） | 代理店向けの詳細システム・機能 |
| 申込者 | 保険の申込者（顧客） |
| eFrance | 保険プロセスを扱う中核システム・プラットフォーム |
| 申込情報・請求書作成 | 申込データ管理と請求書作成 |
| 帳票・契約書発行 | 帳票・契約書の作成・発行 |
| オーナー | 承認・監査役（管理者・コンプライアンス等） |

---

## フロー

### 1. 申込データ送付とURL交換

- GAJ → AGENT（詳細）：氏名、生年月日、電話番号、メールアドレス、物件名、契約番号、プランを送付
- AGENT（詳細）→ eFrance：上記の申込者・プラン情報を転送
- eFrance → 申込情報・請求書作成：同様に転送
- eFrance → 申込者：申込URLを送付
- 申込者 → eFrance：申込情報入力
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

### 2. 見積作成と契約依頼

- GAJ（内部）：承認、見積作成、見積送付
- GAJ → 代理店：見積送付
- 代理店 → GAJ：見積確認
- 代理店 → SUPPLIER：契約
- SUPPLIER → 代理店：承認
- 代理店 → GAJ：契約依頼
- GAJ → AGENT（詳細）：契約情報、プランを送付

### 3. 申込者本人情報の入力用リンク

- AGENT（詳細）→ eFrance：AONT URLを送付
- eFrance → 申込者：本人のリンクを送付
- 申込者 → eFrance：本人情報入力
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

### 4. STEP：保険証券送付（契約・請求書生成を含む）

- GAJ → AGENT（詳細）：申込情報、契約情報、プランを送付
- AGENT（詳細）→ eFrance：同様に転送
- eFrance → 申込情報・請求書作成：同様に転送
- 申込情報・請求書作成 → 帳票・契約書発行：請求書作成を依頼
- 帳票・契約書発行 → オーナー：請求書データを送付
- オーナー → 帳票・契約書発行：承認
- 帳票・契約書発行（内部）：契約書作成
- 帳票・契約書発行 → eFrance：承認済契約書データを送付
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

### 5. STEP：契約情報登録と決済処理

- GAJ → AGENT（詳細）：契約情報、プランを送付
- AGENT（詳細）→ eFrance：クレカ情報を送付
- eFrance → 申込情報・請求書作成：クレカ情報を転送
- 申込情報・請求書作成 → 帳票・契約書発行：日次データと請求書を送付
- 帳票・契約書発行 → オーナー：日次データを送付
- オーナー → 申込情報・請求書作成：承認
- 申込情報・請求書作成 → eFrance：承認済日次データを送付
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

### 6. STEP：メール送付（契約情報等）

- GAJ → AGENT（詳細）：契約情報、プランを送付
- AGENT（詳細）→ eFrance：メールアドレス、契約番号、プランを送付
- eFrance → 申込情報・請求書作成：同様に転送
- 申込情報・請求書作成 → 帳票・契約書発行：同様に転送
- 帳票・契約書発行（内部）：メール作成、メール添付
- 帳票・契約書発行 → オーナー：メールアドレス、契約番号、プランを送付
- オーナー → 帳票・契約書発行：承認
- 帳票・契約書発行 → eFrance：承認済メールアドレスを送付
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

### 7. STEP：メール送付（証券発行通知等）

- 6と同様のパターン（契約情報・プラン、メールアドレス・契約番号・プラン、メール作成・添付、オーナー承認、承認済メールアドレス、AONT URLの転送）

### 8. STEP：メール送付（最終メール送付）

- 6・7と同様のパターン（3回目のメール送付フェーズ）

### 9. STEP：最終承認（契約情報の確定）

- GAJ → AGENT（詳細）：契約情報、プランを送付
- AGENT（詳細）→ eFrance：同様に転送
- eFrance → 申込情報・請求書作成：同様に転送
- 申込情報・請求書作成 → 帳票・契約書発行：同様に転送
- 帳票・契約書発行（内部）：メール作成
- 帳票・契約書発行 → オーナー：契約情報、プランを送付
- オーナー → 帳票・契約書発行：承認
- 帳票・契約書発行 → eFrance：承認済契約情報を送付
- eFrance → AGENT（詳細）：AONT URLを送付
- AGENT（詳細）→ GAJ：AONT URLを転送

---

## 扱う主なデータ

- 氏名、生年月日、電話番号、メールアドレス
- 物件名、契約番号、プラン
- 申込情報、契約情報
- クレカ情報
- 日次データ、請求書、請求書データ
- 承認済契約書データ、承認済日次データ、承認済メールアドレス、承認済契約情報
- 申込URL、AONT URL、本人のリンク

# 2. 火災保険処理_スマラク

シーケンス図「火災保険処理_スマラク」をテキスト化したもの。AGNT・GAファイナンス・箱田・Tnet・eFinance を中心とした申込〜計上確認までの流れ。

---

## 参加者

| 名称 | 説明 |
|------|------|
| AGNT | エージェント（DB・システム） |
| GAファイナンス | GAファイナンス部門 |
| AGNT_進捗表 | エージェント進捗を管理する表・システム |
| 箱田 | 処理担当（個人または役割） |
| 箱田_管理表 | 箱田側の管理用表・システム |
| Tnet | 外部システム |
| eFinance | 外部システム（申込・金融関連） |

---

## フロー

### 1. 情報の受け渡し

- GAファイナンス → AGNT_進捗表：名前、物件名、部屋番号、始期日を送付
- AGNT_進捗表 → 箱田：名前、物件名、部屋番号、始期日を転送
- 箱田 → GAファイナンス：AGNT_URLを送付
- GAファイナンス → AGNT：AGNT_URLを転送
- AGNT → GAファイナンス：謄本・マイソクへのリンクを提供
- GAファイナンス → 箱田：謄本、マイソクを送付

### 2. 申込書と印刷連番の扱い（条件分岐）

**条件：ダミーの印刷連番がない場合**

- 箱田 → eFinance：名前、物件名、部屋番号、始期日（ダミー）を送付
- eFinance → 箱田：申込書を返却
- 箱田（内部）：申込書から印刷連番を取り出す
- 箱田 → eFinance：印刷連番を送付
- eFinance → 箱田：ダミーの印刷連番を返却

### 3. 申込送付とメール

- 箱田 → Tnet：ダミーの印刷連番、名前、物件名、部屋番号、始期日を送付
- 箱田 → eFinance：申込書（クレカ案内付き）＋印刷連番、証券番号、保険料を送付
- eFinance → 箱田：印刷連番、証券番号、保険料を返却
- 箱田 → GAファイナンス：名前、物件名、部屋番号を送付
- 箱田 → GAファイナンス：メールアドレスを送付
- 箱田（内部）：メール作成（7）＋申込書添付
- 箱田 → GAファイナンス：メールを送付
- GAファイナンス → 箱田：メール送信日を返却
- 箱田 → Tnet：申し込み・クレカ登録・計上を依頼

### 4. 申込確認ループ（申し込みが確認できるまで繰り返し）

- 箱田 → Tnet：計上日未登録状況を照会
- Tnet → 箱田：名前、証券番号を返却
- 箱田 → eFinance：証券番号を送付
- eFinance → 箱田：申し込み状況確認を返却
- 箱田 → eFinance：申込状況を報告

**条件：申し込み完了の場合**

- 箱田 → Tnet：計上日を送付

**条件：会社確認が必要な場合**

- Tnet → 箱田：証券番号を返却
- 箱田 → Tnet：計上を依頼
- Tnet → 箱田：計上結果を返却
- 箱田 → Tnet：計上日を送付

- Tnet → 箱田：督促（ループ継続時のフォローとして）

### 5. 完了

- 箱田 → AGNT：完了報告を送付
- **ref クレカ追跡**：下記「3. クレカ追跡」を参照

---

## 扱う主なデータ（スマラク）

- 名前、物件名、部屋番号、始期日
- 謄本、マイソク、AGNT_URL、謄本・マイソクへのリンク
- 印刷連番、ダミーの印刷連番、申込書
- 証券番号、保険料
- メールアドレス、メール送信日
- 計上日、計上日未登録状況、計上結果
- 申し込み状況、申込状況

# 3. クレカ追跡

シーケンス図「クレカ追跡」をテキスト化したもの。クレカ未登録状況の照会から、登録済み／未登録の分岐、メール・SMS送信、オーソリNG時の案内、追跡状況登録まで。

## 参加者

| 名称 | 説明 |
|------|------|
| 箱田 | 処理担当 |
| 箱田_管理表 | 箱田側の管理用表・システム |
| 申込CSV | 申込データ（CSV等） |
| オーナー | 顧客（契約者） |
| Tnet | 外部システム（クレカ・契約情報） |

## フロー

### 1. クレカ未登録状況の照会

- 箱田 → 箱田_管理表：クレカ未登録状況を照会
- 箱田 → 箱田_管理表：証券番号、申込日、名前を提供
- 箱田_管理表 → Tnet：証券番号を送付
- Tnet → 箱田：入金情報、電話番号を返却

### 2. 条件分岐：クレカ登録済み／未登録

**パスA：クレカ登録されていた場合**

- 箱田 → Tnet：入金情報入力
- Tnet → 申込CSV：申込日、名前を送付
- Tnet → 申込CSV：メールアドレスを送付

**パスB：クレカ未登録の場合**

- 箱田 → 箱田_管理表：過去送信分検索
- 箱田 → 箱田_管理表：クレカ登録メール作成
- 箱田 → オーナー：クレカ登録メールを送付
- 箱田 → オーナー：SMS送信

**パスBのうち、オーソリNG（クレカ不備）の場合**

- Tnet → 箱田：証券番号を返却
- Tnet（内部）：クレカ案内作成
- Tnet → 箱田：クレカ案内URLを送付
- 箱田 → 箱田_管理表：クレカ登録案内メール作成（8）
- 箱田 → オーナー：クレカ登録メールを送付
- 箱田 → オーナー：SMS送信

### 3. 分岐後の共通

- オーナー → Tnet：追跡状況を送付
- Tnet（内部）：追跡状況登録
- オーナー → Tnet：クレカ登録

---

## 扱う主なデータ（クレカ追跡）

- 証券番号、申込日、名前、入金情報、電話番号、メールアドレス
- クレカ案内URL、クレカ登録メール、追跡状況

# 4. インシュアランス 付保証明

シーケンス図「インシュアランス 付保証明」をテキスト化したもの。付保証明発行依頼から、証券番号取得・Tnet照会・証明書発行・格納・完了報告まで。

## 参加者

| 名称 | 説明 |
|------|------|
| GAファイナンス | 依頼元 |
| 箱田 | 処理担当 |
| 箱田_ドライブ | 箱田側の共有ドライブ・ストレージ |
| 箱田_スプレッドシート | 箱田側のスプレッドシート（案件・証券番号等） |
| Tnet | 外部システム（契約情報・付保証明発行） |

## フロー

1. GAファイナンス → 箱田：付保証明発行依頼
2. 箱田 → 箱田_ドライブ：格納フォルダ作成（決済月-融資元）
3. 箱田 → 箱田_スプレッドシート：該当案件の証券番号取得を依頼
4. 箱田_スプレッドシート → 箱田：証券番号を返却
5. 箱田 → Tnet：契約情報照会（証券番号）
6. Tnet → 箱田：付保証明書発行
7. 箱田（内部）：付保証明書を受領
8. 箱田 → 箱田_スプレッドシート：証明書発行ステータス変更
9. 箱田 → 箱田_ドライブ：対応する融資元に格納
10. 箱田 → GAファイナンス：完了報告

---

## 扱う主なデータ（付保証明）

- 証券番号、契約情報、付保証明書、証明書発行ステータス、決済月、融資元

---

# 5. インシュアランス 火災保険解約

シーケンス図「インシュアランス 火災保険解約」をテキスト化したもの。オープン通知・解約依頼から、証券番号取得・Tnetでの契約検索・変更依頼書作成・メール送付・書類提出・完了報告まで。

## 参加者

| 名称 | 説明 |
|------|------|
| SUPPLIER | 外部（サプライヤー） |
| GAファイナンス | GAファイナンス部門 |
| AGNT | エージェント（DB・システム） |
| AGNT_スプレッドシート | AGNT側のスプレッドシート |
| INSメール | 保険関連メールシステム |
| 箱田 | 処理担当 |
| eFinance | 外部システム（申込・証券番号等） |
| 箱田_スプレッドシート | 箱田側のスプレッドシート |
| Tnet | 外部システム（契約・変更依頼） |

## フロー

1. AGNT_スプレッドシート → 箱田：オープン通知
2. AGNT_スプレッドシート → 箱田：解約依頼
3. 箱田 → eFinance：証券番号取得を依頼
4. eFinance → 箱田：証券番号を返却
5. 箱田 → Tnet：契約の検索（証券番号）
6. 箱田 → Tnet：変更依頼書の作成を依頼
7. Tnet → 箱田：変更依頼書を返却
8. Tnet → 箱田：変更依頼書＋証券番号、変更後印刷連番、変更後保険料を送付
9. 箱田 → 箱田_スプレッドシート：証券番号、変更後印刷連番、変更後保険料を記入
10. 箱田 → INSメール：変更依頼メール（5,6）変更依頼書＋印刷連番を送付
11. INSメール → 箱田：送信（メール送付）
12. 箱田 → INSメール：変更承諾返信
13. INSメール → AGNT：変更承諾確認
14. AGNT → INSメール：メールPDFを送付
15. eFinance（内部）：計上処理
16. 箱田 → Tnet：書類提出（変更依頼書、承諾メールのPDF）
17. 箱田（内部）：変更完了メール作成（3）
18. 箱田 → （宛先）：メール送信
19. 箱田 → AGNT：完了報告

---

## 扱う主なデータ（火災保険解約）

- 証券番号、変更後印刷連番、変更後保険料、変更依頼書、承諾メールPDF、変更完了メール
