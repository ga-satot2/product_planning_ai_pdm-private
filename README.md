# 個人情報・機密情報ディレクトリ

**⚠️ 重要**: このディレクトリはGitに共有されません。個人情報や機密情報を安全に保存できます。

---

## 📁 ディレクトリ構造

```
private/
├── 1on1_feedback/          # 1on1フィードバック記録
├── personal_notes/         # 個人メモ・ノート
└── confidential/           # 機密情報（M&A情報、人事関連など）
```

---

## 🎯 各ディレクトリの用途

### `1on1_feedback/`
1on1でのフィードバック、成長記録、キャリア開発に関する情報を保存します。

**保存すべき内容**:
- 1on1でのフィードバック記録
- 個人の成長目標・アクションプラン
- キャリア開発に関するメモ
- パフォーマンスレビュー関連のメモ

**保存すべきでない内容**:
- 汎用的なコミュニケーション改善手法（→ `knowledge/communication/` に一般化して保存）

### `personal_notes/`
個人のメモ、学習記録、タスク管理など、個人用の情報を保存します。

**保存すべき内容**:
- 個人の学習メモ
- タスク管理（個人用）
- アイデア・思考メモ
- 個人的なリマインダー

### `confidential/`
機密情報（M&A情報、人事関連情報など）を保存します。

**保存すべき内容**:
- M&A関連情報
- 人事関連の機密情報
- その他、組織の機密情報

**⚠️ 注意**: このディレクトリの内容は、AI（Difyなど）に入力しないでください。

---

## 🔒 セキュリティ

### Git除外設定
このディレクトリはメインリポジトリの `.gitignore` で除外されているため、メインリポジトリには含まれません。

### ローカルGit管理
このディレクトリは**独立したgitリポジトリ**として管理されています。
リモートにプッシュせずに、ローカルでのバージョン管理が可能です。

#### 基本的な使い方

```bash
cd private

# 変更をコミット
git add .
git commit -m "変更内容の説明"

# 履歴確認
git log --oneline

# 変更の確認
git status
git diff

# 過去のバージョンに戻す
git checkout <commit-hash>
```

#### リモートリポジトリを設定する場合（オプション）

将来的にプライベートリモートリポジトリにプッシュしたい場合：

```bash
git remote add origin <your-private-repo-url>
git push -u origin main
```

**注意**: リモートを設定しない限り、`git push`はエラーになりますが、ローカルでの管理は問題なく動作します。

### バックアップ
個人情報・機密情報のバックアップは、適切なセキュリティ対策を講じた場所で管理してください。

### AIへの入力禁止
以下の情報は、AI（Difyなど）に入力しないでください：
- 1on1情報
- M&A情報
- 人事関連情報
- 顧客の個人情報

詳細は `knowledge/technical/security/2025-11-12_codebase_security_risk_assessment.md` を参照してください。
